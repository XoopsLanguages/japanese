ヘルプ: プロフィール

説明

プロフィールモジュールは、カスタムユーザープロフィールフィールドを管理するためのものです。

インストール/アンインストール

特別な手順は必要ありません。標準のインストール手順に従ってください。/profileフォルダを../modulesディレクトリに展開します。管理者 -> システムモジュール -> モジュールからモジュールをインストールします。

モジュールのインストールに関する詳細な手順は、XOOPS運用マニュアル に記載されています。

操作手順

このモジュールをセットアップするには、以下を行う必要があります：

  • モジュールの設定（「設定」を参照）を行い、必要に応じてプロフィールブロックを使用する場合はその設定を行います（「ブロック」を参照）。
  • 既存のカテゴリを編集するか、新しいカテゴリを追加します。
  • 既存のフィールドを編集するか、新しいフィールドを追加します。ここでは、どのフィールドがどのカテゴリに表示されるか、ユーザー登録時に表示されるかを指定できます。
  • 登録ステップの順序を定義します。
  • 最後に、個々のフィールドに対して権限を設定できます。どのフィールドが編集可能か、検索可能かを指定します。

スパム対策

スパム登録を最小限に抑えるために、以下を行ってください：

  • 管理者でプロテクターモジュールに移動し、設定に進み、以下のオプションを確認します： 「Stop Forum Spam」 www.stopforumspam.comのデータベースに登録されているスパマーに対してPOSTデータをチェックします。
    これを「IPを禁止（制限なし）」に設定します。

  • /class/captcha/config.php で、モードが「text」に設定されていることを確認してください：

    return $config = array(
    'disabled' => false, // CAPTCHAを無効にする
    'mode' => 'text', // デフォルトモード。'text'、'image'、'recaptcha'（APIキー必要）から選択可能
    'name' => 'xoopscaptcha', // CAPTCHA名
    'skipmember' => true, // メンバーに対してCAPTCHAチェックをスキップ
    'maxattempts' => 10, // セッションごとの最大試行回数
    );

  • プロフィールモジュールで、管理者に移動し、以下の基本ステップで：
    /modules/profile/admin/step.php?id=1
    「ステップ後の保存」を「いいえ」に設定します。


  • プロフィール設定で、「2番目の登録ステップ後にCAPTCHAを使用する」オプションを「はい」に設定します（デフォルト設定）。
