ヘルプ: テンプレート

説明

XOOPSでは、サイトの実際の表示（「スキン」またはユーザーインターフェース）と背後で動作するコードをある程度分離することができます。前者は、グラフィカルなテーマに基づく高度な仕組みによって処理されます。テーマには、基本的な形式として、サイトの視覚的インターフェースを定義する必須のHTMLテンプレート、スタイルシート、そして視覚的インターフェースを補完するために必要な画像が含まれることがあります。テーマは管理インターフェースで切り替え可能で、XOOPSサイト全体の外観を変更するには、セレクターの値を変更して「承認」をクリックするだけです。XOOPSでは、サイト管理者がユーザーに利用可能なテーマの中から選択させる機能も提供しています。

一般的に言えば、テーマはサイトの視覚的な高レベルな枠組みです。たとえば、テーマはサイトの全体的なレイアウトを定義しますが、ブロック内のコンテンツの具体的な配置は通常定義しません。これは、テンプレートセットと呼ばれる別のエンティティによって処理されます。テンプレートセットは、XOOPSウェブサイトを構成するすべてのブロック内のコンテンツの表示を定義するHTMLテンプレートの集合です。より明確な比喩を使うなら、テーマは家の建築家のようなもので、テンプレートセットはインテリアデザイナーのようなものです。建築家は壁を設置し、全体の配置を定義しますが、壁を「飾る」のはインテリアデザイナーの役割です。

アクティブなテーマがテーマファイル、CSSファイル、画像のみを含むシンプルなものである場合、XOOPSはデフォルトのテンプレートセットを使用してブロックの表示を定義します。ただし、一部の複雑なテーマは独自のテンプレートセットを含むほど高度です。この場合、サイト管理者は管理インターフェースを使用してテーマとテンプレートセットの両方を設定する必要があります。

テーマの「魔法」はどのように実現されるのでしょうか？つまり、HTMLテンプレートに動的なコンテンツをどのように含めるのでしょうか？これは鏡や煙を使ったものではなく、Smartyと呼ばれる賢いテンプレートエンジンによって実現されます。これは、かつて変数補間と呼ばれていた機能を実行します。簡単に言えば、XOOPSシステムはPHPファイルを使用してモジュールのロジック（データベースへのアクセス、変数の割り当て、権限の管理など）を保持し、Smarty変数が挿入されたHTMLファイルを使用して実際の視覚的インターフェースを管理し、Smartyエンジンを使用してエンドユーザーが見るページを構築するHTML内の特定の場所に値や文字列を挿入します。全体として、これはあなたにとって簡単にまとめられます：XOOPSのテーマシステムのおかげで、サイトデザイナーは内部の仕組みをあまり気にせずに、より魅力的で視覚的に美しいサイトを構築できます。

テンプレートセットのクローンとダウンロード

カスタムテンプレートセットを作成したい場合、デフォルトセット（または他のセット）を「クローン」リンクを選択してコピーできます（新しいセットに名前を付けるよう求められます）。デフォルトとは異なり、他のテンプレートセットは自由に編集可能です。また、「ダウンロード」リンクを選択してテンプレートセットをディスクに保存したり、「削除」リンクで削除したりできます。新しいモジュールでテンプレートがまだ作成されていない場合、「生成」リンクが追加で表示されることもあります。

テンプレートの編集

ディレクトリツリービューを参照することで、利用可能なテンプレートの完全なリストを確認できます。テンプレートを選択すると、XOOPSはエディターでそれを開き、編集できます。

テンプレートを編集する際は、現在のテーマのthemes/your theme/templatesディレクトリに新しいファイルがある場合に、モジュールのテンプレート .tpl ファイルを自動的に更新する設定を有効にする必要があります（システムモジュール -> 設定 -> 一般設定 -> モジュールのテンプレート .tpl ファイルをthemes/your theme/templatesディレクトリから更新しますか？ -> 「はい」）。サイトが公開されたら、この設定はオフにする必要があります。

テンプレートを誤って壊してしまった場合、デフォルトのテンプレートセットから「正常な」コピーを取得できることを忘れないでください。標準以外のテンプレートを扱う場合は、変更を始める前にディスクにコピーを保存しておくとよいでしょう。
